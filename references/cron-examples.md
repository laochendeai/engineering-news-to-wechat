# Cron Examples

## 主任务：19:00 生成并上传

- schedule: `0 19 * * *`
- 时区：`Asia/Shanghai`
- 内容：生成日报 Markdown → dry-run → 上传微信公众号草稿箱

## 兜底任务：19:20 巡检并自动补跑

- schedule: `20 19 * * *`
- 时区：`Asia/Shanghai`
- 内容：检查 19:00 主任务是否成功；若失败/超时/无 media_id，则强制补跑一次

## 设计建议

- 主任务只做主流程
- 补跑任务只负责巡检与重试
- 成功时尽量静默
- 失败时明确回报阻塞点
